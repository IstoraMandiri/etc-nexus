# ECIP-1121 Implementation Plan

**Status:** Planning
**Target:** core-geth (IstoraMandiri/core-geth fork)
**Branch:** `ecip-1121` (to be created in core-geth submodule)
**ECIP:** [ECIP-1121 - Execution Client Specification Alignment](https://ecips.ethereumclassic.org/ECIPs/ecip-1121)

## Overview

ECIP-1121 aligns Ethereum Classic's execution layer with Ethereum's Pectra/Fusaka upgrades while maintaining ETC's Proof-of-Work consensus. This document tracks which EIPs need implementation in core-geth.

### Explicitly Excluded (Not Applicable to ETC)

These are excluded from ECIP-1121 because ETC doesn't use these features:

- **Blob-related EIPs** (EIP-4844, EIP-4788, etc.) - ETC doesn't implement blob data availability
- **Proof-of-Stake EIPs** - ETC remains Proof-of-Work
- **Fee market governance** - ETC maintains different economic model
- **EIP-7642** (eth/69 History Expiry) - Removes total difficulty from protocol; incompatible with ETC's PoW architecture ([dropped in ECIPs#563](https://github.com/ethereumclassic/ECIPs/pull/563))

## Implementation Plan

### Phase 0: ETC Test Suite Buildout

Before implementing new EIPs, expand the Hive `consensus-etc` test suite to cover existing ETC forks that don't yet have test coverage:

- [ ] Audit which ETC forks have tests in Hive (Frontier through Magneto/Spiral)
- [ ] Generate or port missing test fixtures for ETC-specific forks (e.g. Atlantis, Agharta, Phoenix, Magneto, Spiral)
- [ ] Ensure baseline test coverage exists so regressions from new EIP ports can be detected
- [ ] Validate all three ETC clients (core-geth, besu-etc, nethermind-etc) pass existing suite before changes

### Phase 1: Low-Complexity Gas/Protocol Changes

These can be implemented first as they are straightforward parameter changes:

#### 1.1 EIP-7623: Increase Calldata Cost
- **Change:** Increase calldata cost to reduce max block size
- **Files:** `core/vm/gas_table.go`, `params/protocol.go`
- **Effort:** Small

#### 1.2 EIP-7825: Transaction Gas Limit Cap
- **Change:** Cap transaction gas at 16,777,216 (2^24)
- **Files:** `core/tx_pool.go`, `core/types/transaction.go`
- **Effort:** Small

#### 1.3 EIP-7883: MODEXP Gas Cost Increase
- **Change:** Adjust gas costs for MODEXP precompile
- **Files:** `core/vm/contracts.go` (modexp pricing)
- **Effort:** Small

#### 1.4 EIP-7935: Default Gas Limit 60M
- **Change:** Update default client gas limit
- **Files:** `params/protocol.go`, client configs
- **Effort:** Trivial

#### 1.5 EIP-7934: RLP Block Size Limit
- **Change:** Add 10 MiB RLP block size validation
- **Files:** `core/types/block.go`, RLP decoding
- **Effort:** Small

### Phase 2: Medium-Complexity Features

#### 2.1 EIP-2935: Historical Block Hashes in State
- **Change:** Store last 8192 block hashes in a system contract
- **Files:**
  - New system contract at designated address
  - `core/state_processor.go` - update hash storage
  - `core/vm/evm.go` - BLOCKHASH opcode modification
- **Reference:** go-ethereum Prague implementation
- **Effort:** Medium

#### 2.2 EIP-7951: secp256r1 Precompile
- **Change:** Add P-256 (secp256r1) ECDSA verification precompile
- **Files:**
  - `core/vm/contracts.go` - new precompile
  - `crypto/` - P-256 operations (may use stdlib)
- **Effort:** Medium

#### 2.3 EIP-7910: eth_config JSON-RPC
- **Change:** Add new RPC endpoint for fork configuration
- **Files:**
  - `internal/ethapi/api.go` or `eth/api.go`
  - New endpoint implementation
- **Effort:** Medium

### Phase 3: High-Complexity Features

#### 3.1 EIP-7702: Set EOA Account Code
- **Change:** Allow EOAs to temporarily set contract code via new transaction type
- **Files:**
  - `core/types/transaction.go` - new tx type 0x04
  - `core/types/transaction_signing.go` - Prague signer
  - `core/state_transition.go` - execution logic
  - `core/tx_pool.go` - validation
  - `core/vm/evm.go` - execution context
- **Reference:** go-ethereum v1.15.0 (Pectra)
- **Effort:** High - This is the most complex EIP

## Implementation Strategy

### Recommended Approach: Port from go-ethereum

All target EIPs are implemented in upstream go-ethereum (Pectra in v1.15.x, Fusaka in v1.16.x). The cleanest approach is:

1. **Create branch** in IstoraMandiri/core-geth: `ecip-1121`
2. **Sync with upstream go-ethereum** to get Prague implementations
3. **Adapt for ETC:**
   - Use block-based activation instead of timestamp
   - Exclude blob-related code paths
   - Add ETC-specific fork configuration
4. **Test with Hive** using our `consensus-etc` suite

### Alternative: Cherry-pick Specific Commits

If full sync is too disruptive:
1. Identify specific commits in go-ethereum that implement each EIP
2. Cherry-pick and adapt each one individually
3. More work but more controlled

## Testing Requirements

1. **Unit tests** for each EIP implementation
2. **Hive consensus tests** - run full `consensus-etc` suite
3. **Mordor testnet** deployment before mainnet
